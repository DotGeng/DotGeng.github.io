java中final和static总结

一:final和static修饰变量
1 其实不用那么花哨的解释，其实final就是规定，其修饰的基本类型数据不能再更改，对于引用类型变量不能再更改引用。
static修饰的变量规定其修饰的变量在内存中只有一份。
2 final和static修饰的变量都存在于java内存模型的运行时常量池中。
3 fianl 修饰的变量在定义的时候，必须要进行初始化。
二 final和static修饰方法
1 final修饰的方法，表示其不能被子类重写；
2 static 修饰的方法，不依赖于任何实例对象，static修饰的
方法必须实现。
三 static 和final 修饰的类
1 final修饰的类，表名该类不能被继承，也就是说该类的设计不能再被做任何修改，或者出于安全的角度考虑，该类不应该有子类。
2 static 一般不用来修饰类，如果一定要用static修饰的话，那么通常static修饰匿名内部类。（请看其他笔记）；
四 ： static 修饰代码块：
通过static关键字用来形成静态代码块，以优化程序性能。static块可以置于类中的任何地方，类中可以有多个static块。在类初次被加载的时候，会按照static块的顺序来执行每个static块，并且只会执行一次，即静态代码块随着类的加载而执行，只执行一次，并优先于主函数执行。
